    Topic 3.1 - 数据可视化¶

    1. 数据可视化的概念¶

    数据可视化的概念大家都不陌生了，简单来说就是画图，用图表来展示数据中的规律和特征：

    • 有一句古话说得好，“不识庐山真面目，只缘身在此山中”
    • 在数据分析的过程中，如果只关注数据的一些描述性特征，例如均值、方差、相关系数等，可能会忽略数据中的一些重要信息，例如数据的分布、异常值、趋势等
    • 通过数据可视化，我们可以更直观地了解数据的结构和特征，发现数据中的规律和趋势，从而更好地进行数据分析和建模
    • 这就是从关注数据的特征到关注数据的结构的转变

    我们来通过一个例子来看一下数据可视化的重要性：

    • 首先，在 R 中有一个包叫做 datasauRus，它当中有个很有趣的数据集叫做 datasaurus_dozen
    • 这个数据集包含了 13 个不同的数据集，每个数据集都有相同的均值、方差、相关系数等描述性统计特征
    • 我们先来加载一下这个数据集，并在其中随便查看两个数据集：

dino_doz$dataset <- factor(dino_doz$dataset, levels = unique(dino_doz$dataset))

    • 可以看到每个数据集中的 X 和 Y 的均值、方差、相关系数等都是一样的
    • 如果不进行进一步的探索，我们可能会总结为这 13 个数据集是一样的，或者说它们之间没有什么区别
    • 但是如果我们对这 13 个数据集进行可视化，我们就会发现它们的结构和特征是完全不同的：
    library(ggplot2)

ggplot(dino_doz, aes(x = x, y = y)) +
geom_point() +
facet_wrap(~ dataset, ncol = 4) +
theme_bw()

    • 通过这个例子我们可以看到，仅仅关注数据的描述性统计特征是远远不够的，我们还需要关注数据的结构和特征，这就是数据可视化的重要性所在

    2. R 中的数据可视化¶

    在我们的课程中，我们会主要使用 ggplot2 包来进行数据可视化：

    • ggplot2 是 R 中最流行的数据可视化包之一，它提供了一个非常灵活和强大的语法来创建各种类型的图表，基本刚刚接触 R 中的数据可视化都要使用 ggplot2 包
    • 但是 ggplot2 包有一个最大的问题就是交互性比较差，创建的图表是静态的，无法进行交互式的探索和分析

    • 还有很多其他的包也可以用来进行数据可视化，并且强调了交互性，例如：

      • lattice 包：提供了一个基于格子的图形系统，可以用来创建复杂的多面板图表
      • plotly 包：提供了一个基于 JavaScript 的交互式图形系统，可以用来创建交互式的图表
      • ggvis 包：提供了一个基于 ggplot2 的交互式图形系统，可以用来创建交互式的图表

    • 这些包大家可以根据自己的需求和喜好来选择使用，当然在我们的课程中我们会主要使用 ggplot2 包来进行数据可视化
